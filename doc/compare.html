compare(3) FreeBSD Library Functions Manual compare(3)

NAME

eq, gt, lt, ge, le,

LIBRARY

libcompare, -lcompare

SYNOPSIS

use compare

logical

eq(real|real64|complex r1|z1, real|real64|complex r2|z2, [real t]);

logical

gt(real|complex r1|z1, real|complex r2|z2, [real t]);

logical

lt(real|complex r1|z1, real|complex r2|z2, [real t]);

logical

ge(real|complex r1|z1, real|complex r2|z2, [real t]);

logical

le(real|complex r1|z1, real|complex r2|z2, [real t]);

DESCRIPTION

The compare family of functions compares real or complex numbers within a tolerance, t as described below. The eq() function checks for equality between the two arguments and returns .true. if the arguments are equal and .false. if they are not. These functions aid in the comparison of floating point numbers. Arguments r1, r2 or z1, z2 must be of same type. If array arguments are supplied to eq() the arguments must match in type and size.

The optional argument t may be used to tune the tolerance used in the comparison. By default, this tolerance is 1e-8 (0.00000001).

eq() accepts arguments of types real , real64 , or complex real or complex

RETURN VALUE

All function return .true. when the condition is true and .false. when the condition is false.

EXAMPLES

A simple example checking equality of two real numbers and printing the result to standard out:

program test_compare
use compare, only: eq
implicit none

write(*,*) eq(1.0,1.0)
write(*,*) eq(2.0,1.0)
end program test_compare

Comparing real64 values with a non-default tolerance:

program test_compare
use compare, only: eq
use iso_fortran_env, only: real64
implicit none

write(*,*) eq(1.0,1.0001,1e-2)
write(*,*) eq(1.0_real64,1.0001_real64,1e-3)
end program test_compare
